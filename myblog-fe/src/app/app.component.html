Sass do 0 ao I

O que é SASS?
Sass é uma linguagem de script que é interpretada ou compilada em Cascading Style Sheets (CSS). SassScript é a linguagem em si. Sass consiste em duas sintaxes. A sintaxe original, chamada de "sintaxe indentada", usa uma sintaxe semelhante a Haml. Ela usa indentação para separar blocos de código e caracteres de nova linha para separar regras. A sintaxe mais recente, "SCSS", usa formatação de bloco, como a de CSS. Esta usa chaves para designar blocos de código e ponto-e-vírgula para separar linhas dentro de um bloco. Os arquivos com sintaxe de indentação e SCSS são tradicionalmente dados as extensões .sass e .scss, respectivamente.

Como funciona o SASS?

O Padrão BEM

O padrão BEM é uma convenção de escrita para class CSS que segue o príncipio Blocks, Elements and Modifiers. O bloco sendo o componente principal, o elemento a parte interna a esse bloco e por fim o modificador é o subconteudo desse elemento como no exemplo a baixo. Como fica o Html: 


  <--- page.hrml --- >

    <div class="bloco">
      <p class="bloco__elemento">  
      Meu conteudo com um  
      <a href="#" class="bloco__elemento--modificador"> link <a>.
     </p>
    </div>

Meu conteudo com um link.

O código em SCSS

Sequindo o padrão BEM com a sintaxe do sass temos os seguinte resultado. 

 
  <--- page.scss --- >    

  @import "src/styles.scss";    
  .bloco {
    background-color: $color-secundary;
    border-left: .3rem solid $color-primary;
    &__elemento {
      font-size: 16px;
      color: purple;
      font-weight: 600;
      &--modificador {
        color: pink;
      }
    }
  }

Variáveis Sass

As variáveis Sass são bem simples de usar: você atribui um valor a um nome que começa com $ e, em seguida, pode se referir a esse nome em vez do próprio valor. Mas, apesar de sua simplicidade, eles são uma das ferramentas mais úteis que Sass traz para a mesa. As variáveis tornam possível reduzir a repetição, fazer matemática complexa, configurar bibliotecas e muito mais. Neste caso estamos armasenando nossas variáveis em um arquivo global e importando o mesmo quando precisamos de alguma variável. 

 
  <--- style.scss --- >   

      $color-primary: #e400be;
      $color-secundary: #f4f5f6;

Porque usar o SASS?

Com o SASS é possível deixar o trabalho de criação de folhas de estilo mais dinâmico, profissional e rápido. Com a ajuda de um compilador deixamos nosso arquivo css sempre compactado melhorando o desempenho do site por carregar folhas de estilo mais leves e de quebra ainda deixa nossa área de trabalho mais organizada por permitir dividir o arquivo em várias partes distintas, realmente para um ambiente de produção o SASS faz muita diferença.
